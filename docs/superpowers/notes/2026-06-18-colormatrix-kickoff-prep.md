## 已验证结论（颜色保真实验）
- **无损 PNG 往返**：4/16/32/64 色深，所有单元格 ≥3px，误判率 **0.000%**。
- **跨机色偏/亮度（gamma 1.1/0.9、+15R 色偏）**：**0%** — 我此前最大担心被证伪（调色板高 ΔE，最近色匹配天然鲁棒）。
- **唯一威胁 = JPEG 重压缩**（色越多越脆）；**强制 PNG 即可规避**（现在本就是 PNG）。
- **2px 单元格对缩放太脆**（8色/2px 在 resize95% 下 12% 误判）→ 单元格 **≥3px**。

## 已确定的决策
1. **新模式，与现有 QR 共存**（不覆盖现有 `--mode single/array`）。
2. **新模式默认参数 = 稳健甜点：16 色 / 4px 单元**（~52.5KB/帧，所有测试场景含 JPEG/缩放/色偏均 0% 误判）。激进档（保证 PNG）可 16色/3px ≈ 93KB/帧。
3. **依赖**：Pillow（已有）+ Reed-Solomon（新增 `reedsolo` 纯 Python 库，可进 exe）。
4. **协议**：新二进制分帧载荷（**不用 base64**，省 33%）；自描述帧头（batch/frame-index/total/dims/palette/ECC）；finder 标记定位几何。
5. **复用**：流式编码 + 进度回调（已实现，task 2/3/4 的成果直接接入）。
